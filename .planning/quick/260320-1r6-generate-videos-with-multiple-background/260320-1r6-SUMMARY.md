# Phase quick-260320-1r6 Plan 01: Per-Video Background Selection Summary

**One-liner:** Per-video background selection in storytelling batch, excluding DB-tracked recent clips and within-batch used clips so each generated video uses a unique background.

## What Was Built

Previously `cmd_generate` picked one background clip before the batch loop, so all videos in a `--count N` run used the same gameplay footage.

Now:
- `cmd_generate` sets `background = None` for the automatic path (only calls `_pick_background_interactive()` when `--pick-background` flag is set).
- Inside `_generate_storytelling_from_backlog`, a `batch_used: list[str]` is initialized before the loop.
- For each video, if no override background is set, the function queries `get_recent_backgrounds()` (DB-tracked) and merges it with `batch_used` into a deduped exclude list, then calls `_pick_background(exclude=...)`.
- After a successful video is produced, `batch_used` receives the chosen clip's basename so subsequent videos in the same batch exclude it.
- The `--pick-background` interactive override is unchanged: user selects one clip, it applies to all videos in the batch.
