# Quick Task 260320-25z: --multi-bg flag — Context

**Gathered:** 2026-03-20
**Status:** Ready for planning

<domain>
## Task Boundary

Add a `--multi-bg` flag to the `generate` command (storytelling format). When set, the user picks multiple background clips interactively, TTS + subtitles are generated once, and one output video is assembled per selected background — all sharing the same run directory.

</domain>

<decisions>
## Implementation Decisions

### Background Selection UI
- Interactive numbered list (same style as `--pick-background`)
- User types comma-separated numbers, e.g. `1,3,5`
- Must select ≥ 2 backgrounds (single selection should use normal flow)

### Output Naming
- All variants share one timestamped run dir
- Files named `final_bg1.mp4`, `final_bg2.mp4`, etc. (1-indexed, matching selection order)
- Shared artifacts (narration.mp3, subtitles.ass, timestamps.json) stay in the same dir

### Backlog Interaction
- Consumes **1 backlog item** — one story is pulled and marked used once
- Produces N output paths (one per background), all returned in the `produced` list

### Claude's Discretion
- CLI flag name: `--multi-bg` (boolean flag, triggers interactive selection)
- `--multi-bg` and `--pick-background` are mutually exclusive (if both set, `--multi-bg` wins)
- `--multi-bg` only applies to storytelling format; for tweets format, ignore silently or warn

</decisions>

<specifics>
## Specific Ideas

- Reuse `_pick_background_interactive()` style list rendering but allow multi-select input
- New helper: `_pick_backgrounds_interactive() -> list[str]` — returns list of selected paths
- In `_run_storytelling_pipeline`, split into two parts: `_prepare_storytelling_assets()` (TTS + subs) and reuse assets for each background assembly
- Output filenames: `final_bg1.mp4`, `final_bg2.mp4`, … (not zero-indexed)

</specifics>
