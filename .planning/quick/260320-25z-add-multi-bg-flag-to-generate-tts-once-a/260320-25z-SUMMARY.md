# Quick Task 260320-25z: --multi-bg Flag — Summary

**One-liner:** Added --multi-bg flag to generate command so TTS runs once per story and one video is assembled per selected background, all sharing a single timestamped run directory.

## What Was Built

### --multi-bg CLI Flag (`main.py`)

Added `--multi-bg` as a `store_true` flag on the generate subparser. When both `--background` and `--multi-bg` are set, `--multi-bg` wins (per CONTEXT.md decision). The flag is passed to `cmd_generate()` as `multi_bg=args.multi_bg`.

### `_pick_backgrounds_interactive()` (`commands/generate.py`)

New helper that reuses the same clip-listing logic as `_pick_background_interactive()` but accepts comma-separated input (e.g. `1,3,5`). Enforces >= 2 selections — prints a usage hint and calls `sys.exit(1)` if the user picks fewer. Returns a deduplicated list of selected background paths in selection order.

### `_assemble_multi_bg()` (`commands/generate.py`)

New helper that:
1. Creates one timestamped workdir
2. Generates TTS once (or silent audio if `--no-audio`)
3. Generates ASS subtitles once
4. Loops over each background, calling `assemble_video()` with output `final_bg{i}.mp4` (1-indexed)
5. Returns list of successfully assembled paths; per-background failures are logged and skipped

### Updated `_generate_storytelling_from_backlog()` (`commands/generate.py`)

Added `backgrounds: list[str] | None = None` parameter. When provided:
- Calls `_assemble_multi_bg()` instead of `_run_storytelling_pipeline()`
- Calls `_save_video_metadata()` for each output path
- Marks the backlog item used once (not per background)
- Logs background use for each background clip chosen
- Extends `produced` list with all variant paths

When `backgrounds` is None, the original single-bg path is unchanged.

### Updated `cmd_generate()` (`commands/generate.py`)

Added `multi_bg: bool = False` parameter. When `multi_bg=True` and format is storytelling, calls `_pick_backgrounds_interactive()` then passes `backgrounds=` to `_generate_storytelling_from_backlog`. When `multi_bg=True` and format is tweets, logs a warning and continues with the normal tweet flow.
