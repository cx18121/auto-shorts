Three bugs to fix in formats/storytelling/assembler.py:

**Bug A — Missing `import json`:**
`_probe_video_duration` calls `json.loads()` but `json` is never imported. This causes
`_random_bg_start` to always catch a NameError and return 0.0. Add `import json` to the
imports at the top of the file (after `import random`).

**Bug B — `_random_bg_start` returns 0.0 for short clips:**
Lines 209-210: when `bg_duration <= required_duration`, the function returns 0.0. Since the
clip is stream-looped (`-stream_loop -1`), any start offset works fine — the video wraps.
Replace the early return with `random.uniform(0, bg_duration)` so even short clips get a
random start. Update the log message to note it is a looped clip. The full branch should be:
```python
if bg_duration <= required_duration:
    start = random.uniform(0, bg_duration)
    logger.info("Random background start: %.2fs (clip=%.2fs, looped)", start, bg_duration)
    return start
```

**Bug C — `assemble_split_video` / `_build_split_ffmpeg_cmd` never randomizes start:**
1. In `assemble_split_video`, compute the adjusted duration and call `_random_bg_start`
   before building the command, exactly like `assemble_video` does:
   ```python
   adjusted_duration = duration_seconds / AUDIO_SPEED + 0.5
   bg_start = _random_bg_start(bg, adjusted_duration)
   ```
   Pass `bg_start` to `_build_split_ffmpeg_cmd`.

2. Add a `bg_start: float = 0.0` parameter to `_build_split_ffmpeg_cmd`.

3. In the return list inside `_build_split_ffmpeg_cmd`, insert `"-ss", str(bg_start)`
   before the `"-stream_loop"` line, matching the pattern used in `_build_ffmpeg_cmd`.
