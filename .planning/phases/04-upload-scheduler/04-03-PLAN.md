    **Part B: Implement cmd_run_cycle(channel_cfg)**

    This is the core orchestrator. Flow:

    1. Check `channel_cfg.enabled` — if False, log info "Channel {slug} is disabled, skipping" and return.
    2. Lazy import: `from analysis.db import get_connection`, `from pipeline.backlog import get_approved_stories, get_approved_tweets, mark_story_used, mark_used`, `from pipeline.upload import upload_to_youtube, upload_to_instagram, generate_upload_metadata, init_upload_table, log_upload, refresh_instagram_token_if_needed`.
    3. Open DB connection. Call init_upload_table(conn) to ensure table exists.
    4. Determine format from channel_cfg.format.
    5. **Pick best item from backlog:**
       - If storytelling: `rows = get_approved_stories(conn, channel_cfg.slug)`. If empty, call `cmd_scrape("reddit", "week", channel_cfg)` as fallback, then re-query. If still empty, log warning "No approved items for {slug} after scrape fallback" and return.
       - If tweets: `rows = get_approved_tweets(conn, channel_cfg.slug)`. If empty, call `cmd_scrape("tweets", "week", channel_cfg)` as fallback, then re-query. If still empty, log warning and return.
       - Take `rows[0]` (highest scored/liked).
    6. **Generate video:**
       - For storytelling: Use the same flow as _generate_storytelling_from_backlog but for exactly 1 item. Call adapt_reddit_post, _generate_with_quality, _run_storytelling_pipeline. Get video_path.
       - For tweets: Build tweet dict from row, call _run_tweet_pipeline. Get video_path.
       - If video generation fails, log error and return.
    7. **Generate upload metadata:**
       - Extract content_text from the item (story_text for storytelling, tweet_text for tweets).
       - Call generate_upload_metadata(content_text, channel_cfg.hashtags, channel_cfg.format).
       - Get title, description (template: "{title}\n\n#{' #'.join(all_hashtags)}"), tags (merged hashtags).
    8. **Upload to YouTube:**
       - Compute yt_token_path = Path(config.CHANNELS_DIR) / channel_cfg.slug / "youtube_token.json".
       - If yt_token_path does not exist, log WARNING "No YouTube token for {slug} — skipping YouTube upload" and skip.
       - Otherwise: try upload_to_youtube(video_path, title, description, tags, yt_token_path, channel_cfg.youtube_client_id, channel_cfg.youtube_client_secret). Log success. Call log_upload(conn, slug, "youtube", video_id, title, "success").
       - On exception: log ERROR. Call log_upload(conn, slug, "youtube", "", title, "failed", error_msg=str(e)).
    9. **Upload to Instagram:**
       - Compute ig_token_path = Path(config.CHANNELS_DIR) / channel_cfg.slug / "instagram_token.json".
       - If ig_token_path does not exist or channel_cfg.instagram_user_id is empty, log INFO "Instagram not configured for {slug} — skipping" and skip.
       - Otherwise: call refresh_instagram_token_if_needed(ig_token_path) to get current access_token. Build caption = "{title}\n\n#{' #'.join(all_hashtags)}".
       - NOTE: Instagram requires a publicly accessible video_url. If INSTAGRAM_PUBLIC_BASE_URL is set in environment, construct URL = f"{base_url}/{os.path.basename(video_path)}". Otherwise log WARNING "INSTAGRAM_PUBLIC_BASE_URL not set — skipping Instagram upload" and skip.
       - try upload_to_instagram(video_url, caption, channel_cfg.instagram_user_id, access_token). Log success. Call log_upload(conn, slug, "instagram", media_id, title, "success").
       - On exception: log ERROR. Call log_upload(conn, slug, "instagram", "", title, "failed", error_msg=str(e)).
    10. **Mark item as used:**
        - For storytelling: mark_story_used(conn, row["id"]). conn.commit().
        - For tweets: mark_used(conn, "backlog_tweets", row["id"]). conn.commit().
    11. Close conn in finally block.
    12. Log summary: "Run cycle complete for {slug}: YouTube={yt_status}, Instagram={ig_status}".
