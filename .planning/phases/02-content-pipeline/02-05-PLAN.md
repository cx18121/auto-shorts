  <name>Task 1: Fix Playwright browser leak with try/finally in _scrape_async</name>
  <files>formats/tweets/scraper.py</files>
  <action>
    In the existing `_scrape_async()` function, wrap the browser's entire lifecycle in try/finally
    so `browser.close()` is guaranteed to be called even if an exception is raised.

    Current structure:
    ```python
    async with async_playwright() as pw:
        browser = await pw.chromium.launch(headless=True)
        ctx = await browser.new_context(...)
        await ctx.add_cookies(cookies)
        page = await ctx.new_page()
        # ... home feed scraping ...
        # ... account loop ...
        await browser.close()
    ```

    Target structure:
    ```python
    async with async_playwright() as pw:
        browser = await pw.chromium.launch(headless=True)
        try:
            ctx = await browser.new_context(...)
            await ctx.add_cookies(cookies)
            page = await ctx.new_page()
            # ... home feed scraping ... (unchanged)
            # ... account loop ... (unchanged)
        finally:
            await browser.close()
    ```

    The `async with async_playwright() as pw:` context manager handles playwright cleanup. The
    try/finally inside it ensures browser.close() runs regardless. Do NOT change any other logic —
    this is a minimal, surgical fix.
